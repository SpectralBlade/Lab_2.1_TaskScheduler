# Источники задач и контракты

## Введение
Была реализована подсистема приёма задач (база) для платформы обработки задач с помощью утиной типизации ("Duck Typing"). Модель задачи Task реализована с помощью `__slots__` (решил попробовать такой подход после того, как узнал о нем на семинаре)

## Схема работы
Пользователь запускает программу через точку входа - main.py. Ему предлагается ввести имя файла со списком задач для парсинга. После ввода, если все успешно (файл существует и в нем есть список задач) происходит парсинг источников, сбор задач с них и параллельная запись в кеш/логи. После обработки источников пользователь может взаимодействовать с интерактивной консолью (CLI) - просмотреть все спаршенные источники, увидеть все задачи или задачи какого-то конкретного источника.

## Пользовательские допущения
Для корректной обработки данных:
- Да, источники задач в данный момент читаются из файла
- Файл с источниками должен находиться по пути src/examples
- Файл должен быть в формате .py
- Источники задач должны храниться в списке с названием SOURCES
- В директории уже есть файл examples.py для теста

Удачного пользования парсером задач!
