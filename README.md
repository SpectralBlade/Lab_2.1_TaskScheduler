# Источники задач и контракты

## Введение
Была реализована подсистема приёма задач (база) для платформы обработки задач с помощью утиной типизации ("Duck Typing"). Модель задачи Task реализована с помощью `__slots__` (решил попробовать такой подход после того, как узнал о нем на семинаре)

## Схема работы
Пользователь запускает программу через точку входа - main.py. Она обрабатывает задачи из src/examples/examples.py и записывает результаты в логи. В логах видны: вся информация по обработанным источникам, их задачам, а также ошибки (если они возникли в процессе).
